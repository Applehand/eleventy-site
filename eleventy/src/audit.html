1.1 Objective & Focus

Objective Focus

  • Navigation structure

1.2 Summary of Main Findings

Search crawlers struggle to determine which pages are important for indexing due to:

  • A large volume of data
  • Inconsistent ranking and navigational signals

Key issues impacting SEO:

  • Inefficient crawl budget allocation: Inconsistent use of Robots.txt and noindex meta tags.
  • Poor URL management: readability, query parameters, and canonicals
  • Slow site speed: ~70% of URLs load at medium or low speed
  • Unoptimized Content & Metadata: Duplicate content and weak keyword targeting

2. Crawlability & Indexability

The site is large and lacks enough contextual signals for crawlers to process effectively.

2.1. Robots.txt & Meta Robots Tags

Robots.txt: Proper robots.txt configuration optimizes crawl budget and helps prevent duplicate content.

  • Optimize Crawl Budget: Block query string URLs to avoid crawling low-value or duplicate pages.
  • Prevent Duplicates: Reduce index bloat by filtering out unnecessary URL variations.
  • Efficient Patterns: Use wildcards and canonical tags to consolidate ranking signals and ensure important pages are prioritized.

Meta Robots (Noindex): Around 1M pages, including some with potential value, have a noindex tag. It's important to ensure these tags are valid and not applied to valuable pages that should be indexed.

2.2. XML Sitemaps

A well-structured XML sitemap is vital for large sites to ensure efficient navigation and accurate indexing.

  • 614k URLs found in a sitemap: 75k indexed; 215k not crawled; 23k non-indexable.
  • 122k indexed URLs missing from the sitemap.
  • 373 URLs return non-200 status codes.

Orphaned Pages

3.1M URLs are not found in the sitemap; 148k have been crawled and are driving traffic.

Additionally, 56k indexed URLs are missing from the sitemap and lack internal links, including 2.1k productive pages.

2.4. Canonical Tags & Duplicate Content

Canonical tags are an essential tool for preventing duplicate content issues. They inform search engines of the preferred version of a page, helping consolidate link equity and improving indexation.

  • 337k pages lack a canonical tag
  • 1.9 million pages point to the "same" canonical
  • 3k pages have an "other" canonical

Many URLs with the "same" canonical may need to be updated to point to a master version to prevent duplicate content issues and consolidate SEO value.

3. Site Speed & Core Web Vitals

~70% of the site has slow or medium page load speeds. Key areas to address:

  • Page Load Performance: Optimize metrics like Largest Contentful Paint (LCP) and Time to First Byte (TTFB).
  • Render-Blocking Resources: Minimize and defer (where possible) unoptimized CSS/JavaScript that delays rendering.
  • Image & Media Optimization: Compress, make responsive, and implement lazy-loading for images.

Consider implementing a Content Delivery Network (CDN) to improve server response times, especially for global users.

Additionally, server and database optimizations, such as caching and query optimization, can further reduce TTFB and improve overall page speed.

4. Mobile-Friendliness & User Experience

Optimizing for mobile is crucial for SEO, as search crawlers prioritize mobile-friendly content and most users access sites via mobile devices.

  • Responsive Design Issues: Address layout shifts. There are 869k URLs with images that lack image height/width attributes.
  • Mobile Page Speed: Assess if mobile performance lags behind desktop and identify improvement areas.
  • User Experience: Evaluate mobile navigation, touch target sizes, and overall usability for a smoother experience.

Leverage the Chrome User Experience Report (CrUX) to gain insights from real-world user experience data.

5. Structured Data & Rich Snippets

No structured data is currently implemented. Implementing structured data can significantly enhance search visibility.

  • Opportunity: Implement JSON-LD schema for key pages to improve SEO performance.
  • Benefits: Rich snippets enhance click-through rates and provide search engines with clearer page context.

6. Internal Linking & Site Architecture

A site of this size requires a robust internal linking hierarchy to help crawlers identify important pages and improve overall crawlability.

Additionally, implementing a breadcrumb trail enhances navigation for both users and crawlers by clearly delineating the site's hierarchical structure.

6.2. Anchor Text Optimization

Ensure internal links use clear, descriptive, and keyword-rich anchor text to boost relevance signals. Perform an audit to identify and improve underperforming links.

6.3. Broken Links & Redirect Issues

336k URLs with internal inbound links are not returning a 200 status.

331k URLs have more than one server redirect rule, which could lead to unnecessary redirects, slowing down page load times.

Resolving these issues will help:

  • Preserve link equity by ensuring crawlers can access important pages without any detours.
  • Enhance the user experience by minimizing unnecessary redirects and preventing broken links.

7. URL Structure & Query Parameters

Enhancing URL clarity can help users and search crawlers better understand a site’s structure and content.

7.1. URL Readability

URLs are currently not short, descriptive, or keyword-friendly. Improving URL readability can enhance user experience and signal relevance to search engines.

7.2. Query String Management

Query string parameters may be creating duplicate content issues. Consider moving these parameters to HTTP headers or POST requests to streamline URLs and improve indexation.

8. Growth Strategy & Prioritization

To reach our goal of increasing traffic by 20%, we will leverage high-impact quick wins, medium-term optimizations, and long-term SEO strategies to drive growth and enhance organic visibility.

8.1. High-Impact Quick Wins

Short-term actions to boost traffic and visibility.

  • Sitemap Optimization: Ensure all important pages are included and properly categorized to enhance crawl efficiency.
  • Noindex Meta Tags: Ensure valuable pages do not have a noindex tag.
  • Meta Data Improvements: Optimize title tags and meta descriptions for improved context and higher click-through rates.
  • Internal Linking Fixes: Address non-200 status internal links to limit navigational detours.
  • Schema Markup Expansion: Begin structured data implementation to improve search comprehension.

8.2. Medium-Term Fixes

Structural improvements that take time and development effort.

  • URL Optimization: Standardize URL structures for clarity and consistency.
  • Query String Refactoring: Minimize reliance on query parameters, using HTTP headers or POST requests where possible.
  • Mobile Usability Enhancements: Improve Core Web Vitals by refining mobile performance and usability.
  • Canonical Optimization: Resolve duplicate content issues by refining canonical tag implementation.

8.3. Long-Term SEO Strategy

Larger initiatives to ensure sustainable growth and improved organic reach.

  • Site Structure & Navigation Redesign: Standardize and flatten site architecture to reduce the average crawl depth and enhance navigation.
  • Content & Keyword Strategy Development: Expand high-value content areas with a focus on evergreen and authority-building topics.
  • Internal Linking Strategy: Establish a robust approach to linking for improved crawlability and page authority distribution.
  • Programmatic SEO Implementation: Use dynamic templates to create consistent landing pages at scale with standardized metadata, headings, and structured data. Use canonical tags and optimized URL structures to avoid duplicate content and ensure key pages are properly indexed.
