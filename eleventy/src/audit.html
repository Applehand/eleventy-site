SEO Tech Audit Demo

Jake Applehans

1.1 Objective & Focus

20% increase in organic traffic YoY

Focus on getting crawled URLs indexed via improving:

  1. Navigation Structure & Average Crawl Depth
  2. Site Speed
  3. Sitemap Index Categories
  4. URL Structure & Query String Management

1.2 Summary of Main Findings

Search crawlers are struggling to understand the site and which pages are important for the index.

The following items are most telling:

  • Poor crawl budget utilization
  • Poor Management of URLs regarding readability, query parameters, and canonicals.
  • Site Speed: ~70% of URLs are slow or medium speed
  • Duplicate Meta Descriptions, H1s, & Titles (content likely as well)

2. Crawlability & Indexability

The site is large and lacks enough contextual signals for crawlers to process effectively.

3.7 million total unique URLs. 2.3 million have been crawled.

908k URLs are indexable, but only 198k URLs are actually indexed.

2.1. Robots.txt & Meta Robots Tags

1.4k indexed URLs are being blocked by robots.txt. Why?

Around 1 million pages, including many with potential value, are blocked from indexing by a noindex tag.

2.2. XML Sitemaps

  • 614k total URLs in sitemap, only 75k indexed. 215k URLs in sitemap not crawled. 23k non-indexable pages in sitemap.
  • 122k URLs are indexed and not in the sitemap.
  • 373 non-200 status URLs in the sitemap.

Reorganize sitemap categories within the sitemap index.

2.3. Crawl Budget

  • Total URLs: 3.7M unique pages
  • Average Crawl Depth: 6.5
  • 2.3 million crawled URLs, only ~200k are indexed
  • 416k crawled URLs that are not 200 status
  • Crawl Frequency
  • Server Performance
  • Dynamic Content
  • Crawl Waste
  • Sitemap Prioritization

2.4. Canonical Tags & Duplicate Content

  • 337k pages have no canonical tag
  • 1.9 million pages have "same" canonical
  • 3k pages have "other" canonical

Pages with the "same" canonical type might need to point to a master version to avoid duplicate content issues

3. Site Speed & Core Web Vitals

~70% of the site is slow or medium speed. Key areas to monitor include:

  • Page Load Performance: Measure metrics like Largest Contentful Paint (LCP) and Time to First Byte (TTFB).
  • Render-Blocking Resources: Identify and optimize unoptimized CSS/JavaScript that delay rendering.
  • Image & Media Optimization: Ensure images are compressed, responsive, and lazy-loaded.

Consider setting up a CDN.

4. Mobile-Friendliness & User Experience

  • Responsive Design Issues: Address layout shifts. There are 869k URLs that lack image height/width attributes.
  • Mobile Page Speed: Assess if mobile performance lags behind desktop and identify improvement areas.
  • User Experience: Evaluate mobile navigation, touch target sizes, and overall usability for a smoother experience.

5. Structured Data & Rich Snippets

No structured data is currently implemented.

  • Opportunity: Implement JSON-LD schema for key page types to enhance search visibility.
  • Benefits: Rich snippets can improve click-through rates and provide clear context to search engines.

6. Internal Linking & Site Architecture

A site of this size needs a strong internal linking strategy to signal to crawlers which pages are important, and to improve overall crawlability.

1.4 million URLs have 0 internal inbound links, 64k of which are indexed without having been crawled.

6.1. Navigation Depth & Link Equity

Key pages require over 3 clicks to access; the average crawl depth is 6.42. Optimizing internal links can distribute link equity better and improve page discoverability.

6.2. Anchor Text Optimization

Ensure internal links use descriptive, keyword-rich anchor text to boost relevancy signals. Audit current links to identify opportunities for improvement.

6.3. Broken Links & Redirect Issues

There are 4,799 URLs returning 404 errors, many with numerous internal links. Streamlining these issues can help preserve link equity and enhance user navigation.

7. URL Structure & Query Parameters

7.1. URL Readability

URLs are currently not short, descriptive, or keyword-friendly. Improving URL readability can enhance user experience and signal relevance to search engines.

7.2. Canonicalization of URLs

While www vs. non-www and HTTP vs. HTTPS are mostly consistent, canonical tag implementation likely needs refinement. Many pages use self-referencing canonicals when consolidation to a master version would be more effective, potentially causing duplicate content issues.

7.3. Query String Management

Query string parameters may be creating duplicate content issues. Consider moving these parameters to HTTP headers or POST requests to streamline URLs and improve indexation.

8. Growth Strategy & Prioritization

8.1. High-Impact Quick Wins

Immediate actions to boost traffic and visibility with minimal effort.

  • Sitemap Optimization: Ensure all important pages are included and properly categorized to enhance crawl efficiency.
  • Meta Data Improvements: Optimize title tags and meta descriptions for improved context and higher click-through rates.
  • Internal Linking Fixes: Address broken or weak internal links to strengthen site architecture.
  • Schema Markup Expansion: Broaden structured data implementation to improve search visibility.

8.2. Medium-Term Fixes

Structural improvements requiring development effort but offering strong ROI.

  • URL Optimization: Standardize URL structures for clarity and consistency.
  • Query String Refactoring: Minimize reliance on query parameters, using HTTP headers or POST requests where possible.
  • Mobile Usability Enhancements: Improve Core Web Vitals by refining mobile performance.
  • Canonical Optimization: Resolve duplicate content issues by refining canonical tag implementation.

8.3. Long-Term SEO Strategy

Larger initiatives to ensure sustainable growth and improved organic reach.

  • Site Structure & Navigation Redesign: Standardize and flatten site architecture to enhance crawl depth and user navigation.
  • Content Strategy Development: Expand high-value content areas with a focus on evergreen and authority-building topics.
  • Internal Linking Strategy: Establish a robust, programmatic approach to linking for improved page authority distribution.
  • Programmatic SEO Implementation: Use dynamic templates to create consistent, high-quality landing pages at scale. Standardize metadata, headings, and structured data, and implement strong internal linking to improve crawl efficiency. Use canonical tags and optimized URL structures to avoid duplicate content and ensure key pages are properly indexed.
